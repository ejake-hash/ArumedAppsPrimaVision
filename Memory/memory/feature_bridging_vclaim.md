**Strategi**: zero-code activation — credential (`cons_id`/`secretKey`/`user_key`) belum turun dari Trustmark. Bangun fondasi + endpoint sekarang, admin tinggal paste di UI → Test → Aktifkan.

**Kondisi AWAL repo (sebelum batch, audit 2026-05-29)**: `IntegrationConfig` model+tabel (credentials cast `array`, belum encrypted), `BpjsVClaimService`+`BpjsAntreanService` (method `placeholder()`), `IntegrasiService` (config CRUD+test+log readers), `IntegrasiController` (21 endpoint `/integrasi/*` aktif), 7 tabel log/data BPJS. Kolom `visits`: `no_sep`,`bpjs_booking_code`,`bpjs_antrean_number`,`bpjs_referral_in_id`,`bpjs_control_letter_id`,`planning_follow_up`/`follow_up_date`. (Status TERKINI ada di bagian "Yang sudah dibuat" di bawah — placeholder sudah diganti implementasi nyata; frontend masih belum ada.)

**Arsitektur disepakati**: kelas baru `App\Services\Bpjs\BpjsClient` (HMAC-SHA256 signature + headers + HTTP + decrypt AES-256-CBC/SHA256 + LZString::decompressFromEncodedURIComponent). VClaim response terenkripsi (`$encrypted=true`), Antrean JSON polos (`false`). Key decrypt = consid+conspwd+timestamp. Library: `nullpunkt/lz-string-php` (composer). `time()` PHP sudah UTC → X-timestamp benar; NTP wajib sinkron.

**Keputusan user (2026-05-29)**: (1) buat semua endpoint backend dulu, (2) fokus VClaim SEP, (3) mapping Poli/DPJP **disinkronkan ke menu Jadwal Dokter**, (4) endpoint RANAP ikut dimasukkan (service lengkap RJ+ranap). Expose service + endpoint lengkap. composer dijalankan.

**Mapping BPJS via Jadwal Dokter**: `doctor_schedules` sudah punya `poli_code` (GLA/EKS/KAT/RET) + `employee_id`. Rencana: `employees`+kolom `bpjs_dpjp_code`; tabel `bpjs_poli_mappings` (poli_code lokal→kode_poli_bpjs). Dropdown isi dari `/referensi/poli` & `/referensi/dokter`. `generateSep` lookup `visit.doctor_schedule_id`→poli_code→kode BPJS + employee→dpjp_code. Antrean wajib sync `/jadwaldokter/...` per minggu dari JadwalDokterView.

**Peta endpoint→UI** (detail di plan §2): Peserta/Rujukan/SEP→AdmisiView; RencanaKontrol→DokterView (nyambung planning_follow_up); LPK→KasirView; Monitoring/Log/Config→IntegrasiView (BARU, 7 tab); Antrean add/updatewaktu→hook QueueService (async non-blocking, lihat [[queue-advance-station-pattern]]); Referensi→cache+autocomplete. RANAP/INA-CBGs grouper/SPRI = prioritas rendah tapi dibuat.

**Guardrail**: call gagal/credential kosong → 503 jelas, log ke bpjs_vclaim_logs/bpjs_antrean_logs, JANGAN blok flow UMUM/non-BPJS. Tidak sentuh modul Asuransi TPA non-BPJS [[feature-insurance-tpa]]. Credential encrypted+masked. Permission baru `integrasi.*`. Styling [[feedback-styling-visibility]], timezone [[feedback-timezone-wib]].
